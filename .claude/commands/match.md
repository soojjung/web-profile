`resume-job-match-evaluator` 에이전트를 사용해 이력서와 채용 공고의 매칭 정도를 평가하세요.

**입력:** $ARGUMENTS

## 처리 방법

1. 사용자 입력에서 **이력서**와 **채용 공고(JD)** 두 가지가 모두 있는지 확인하세요. 둘 중 하나라도 없으면 명확히 어느 쪽이 누락됐는지 알리고 요청하세요.
2. 입력 형태에 따라:
   - **파일 경로**: Read로 읽기
   - **URL (예: linkedin.com, lever.co, greenhouse.io)**: WebFetch로 JD 가져오기
   - **본문 텍스트**: 그대로 사용
3. `resume-job-match-evaluator` 에이전트를 Agent 도구로 실행하되, 프롬프트에 다음을 명시:
   - **이력서 전체 원문**
   - **채용 공고 전체 원문** (회사명/직무명 포함)
   - 8단계 워크플로우를 **모두** 수행할 것 — JD 분석, 키워드 매칭, Gap 분석, ATS 점수, 인터뷰 확률, 권고사항
   - **이력서에 없는 경험을 절대 추론하거나 가정하지 말 것**. 모호하면 모호하다고 표시할 것.
   - 언어: JD가 한국어면 한국어, 영어면 영어로 작성
4. 결과를 사용자에게 전달하고, 가장 critical한 Gap 3개와 즉시 보완 가능한 항목을 강조해 주세요.

## 다음 단계 안내

평가 결과가 부족하다면 `/tailor`로 이력서를 JD에 맞춰 재구성할 수 있다고 안내하세요.
